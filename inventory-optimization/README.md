# inventory-optimization

Inventory allocation module using Linear Programming (LP) and Proportional Allocation to maximise revenue under a stock constraint.

## How it works

Two allocation strategies are computed and compared:

1. **LP Revenue Maximisation** — uses [OR-Tools](https://developers.google.com/optimization) GLOP solver to find the optimal stock allocation that maximises total revenue without exceeding the stock limit
2. **Proportional Allocation (Largest Remainder Method)** — distributes stock proportionally to predicted demand, using the largest remainder method to resolve rounding

## Input

`data/inventory_s001_north_may_2022.csv` — generated by the `demand-forecasting` module. Run that first:

```bash
uv run demand-forecasting
```

Expected columns:

| Column | Description |
|---|---|
| Product | Product identifier |
| Price | Unit price |
| Predicted Demand Forecast | Forecasted demand (from demand-forecasting output) |

## Output

- Results printed to stdout with stock allocations and revenue comparison per strategy
- `data/inventory_optimization_results.csv` — full results with added columns:

| Column | Description |
|---|---|
| LP_Max_Revenue_Stock | Units to stock per LP solution |
| Prop_Stock_LRM | Units to stock per proportional allocation |
| LP_Revenue | Revenue under LP strategy |
| Prop_Revenue | Revenue under proportional strategy |

## Run

From the workspace root:

```bash
uv run inventory-optimization
# or
uv run python -m inventory_optimization
```
