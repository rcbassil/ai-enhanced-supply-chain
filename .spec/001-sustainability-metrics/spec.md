# Spec: Sustainability Metrics Integration

## 1. Objective
Trace and optimize the carbon footprint of the supply chain routing and inventory storage.

## 2. Requirements
- Calculate CO2 emissions for each route generated by `routing-optimization`.
- Estimated carbon cost of storage per product category in `inventory-optimization`.
- Display a "Sustainability Score" in the `query.py` interface.

## 3. User Journeys
- **As a Logistics Manager**, I want to see the total emissions of the optimized route so I can report on ESG goals.
- **As an Inventory Analyst**, I want to compare standard LP allocation with a Carbon-Aware allocation.

## 4. Constraints
- Must not degrade the performance of existing solvers by more than 10%.
- Emission factors should be configurable via a `sustainability_config.json`.
